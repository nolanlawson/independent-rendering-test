Independent Rendering test

Microsoft confidential What is this?

Edge and IE have a feature called Independent Rendering, which allows rendering work to be done off the main thread and instead on a background thread, where it doesn't block JavaScript, DOM events, or other main-thread-bound operations.

In the current implementation, though, some "advanced" features will cause Edge/IE to bail out of Independent Rendering and instead do rendering on the main thread. This will lead to things like lower framerate, delayed JavaScript, choppy rendering, etc.

In this demo, you can add certain "advanced" features that will trigger Independent Rendering deoptimization, and after scrolling through the list the drop in framerate should be fairly obvious. Note that the framerate counter uses requestAnimationFrame(), so it only measures jank for UI-thread-dependent operations, e.g. hardware-accelerated CSS animations will not be janked, but animated GIFs (as in the list below) will be.
